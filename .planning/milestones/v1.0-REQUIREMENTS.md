## V1 Requirements

### Consent and Access

- [x] **CONS-01**: Visitor can read required consent text before submitting conference participation data
- [x] **CONS-02**: Participant can explicitly submit required consent choices with a recorded timestamp and text version
- [x] **AUTH-01**: User can create an account with email, password, and attendance format
- [x] **AUTH-02**: User can log in and stay logged in across sessions
- [x] **AUTH-03**: User can request a password reset by email without exposing whether the email exists in the system
- [x] **AUTH-04**: User can set a new password from a valid reset link

### Registration and Profile

- [x] **PROF-01**: Participant can enter and edit full name, workplace, position, city, academic degree or title, email, and phone number
- [x] **PROF-02**: Participant can choose a predefined conference section during registration or profile completion
- [x] **PROF-03**: Participant can enter and edit the title of their talk
- [x] **PROF-04**: Admin can edit a participant's attendance format, section, and talk title after registration

### Program Management

- [x] **PROG-01**: Admin can assign or edit the final room and time slot for a participant talk in the official conference program
- [x] **PROG-02**: Participant can view their final placement in the conference schedule grid
- [x] **PROG-03**: Official program documents are generated from the authoritative schedule data rather than raw participant profile data
- [x] **PROG-04**: Admin can store an external videoconference link for online sessions or conference participation

### Participation Experience

- [x] **PART-01**: Offline participant can view room and location details for assigned sessions
- [x] **PART-02**: Offline participant can navigate to the correct venue location through a dynamic list or map-based session view
- [x] **PART-03**: Online participant can access an external join link without depending on venue-specific location flows

### Communication

- [x] **CHAT-01**: Participant can send text messages in the conference chat
- [x] **CHAT-02**: Participant can attach allowed files to chat messages
- [x] **CHAT-03**: Participant can download attachments that are available in authorized chat scopes
- [x] **FEED-01**: Participant can submit conference feedback and improvement suggestions

### Documents

- [x] **DOCS-01**: Participant can download a personal conference program in PDF format
- [x] **DOCS-02**: Participant can download a QR badge in PDF format for onsite registration
- [x] **DOCS-03**: Participant can download a certificate in PDF format when participation conditions are met
- [x] **DOCS-04**: Participant can access conference proceedings after the event is completed

### Conference Presentation

- [x] **INFO-01**: Visitor can access public pages about the university, the institute, and the conference
- [x] **INFO-02**: Visitor can see conference branding and key event information consistently across public and authenticated pages
- [x] **UX-01**: User can complete registration, schedule viewing, chat, feedback, and document download on phone, tablet, and desktop layouts

## Deferred to the Next Milestone

### Notifications

- [ ] **NOTF-01**: Participant receives email reminders or updates when schedule details change
- [ ] **NOTF-02**: Participant receives conference-status reminders before key event dates

### Communication Enhancements

- [ ] **CHAT-04**: Participant can preview supported attachment types inline in chat
- [ ] **CHAT-05**: Admin can moderate or remove inappropriate attachments from chat history

### Organizer Enhancements

- [ ] **ADMIN-01**: Organizer can export consent and attendance audit reports
- [ ] **ADMIN-02**: Organizer can edit richer public landing-page content blocks without code changes

## Out of Scope at v1.0

| Feature | Reason |
|---------|--------|
| Multi-conference multi-tenant SaaS administration | First release remains for one conference only |
| Embedded video conferencing inside the platform | Online participants only need an external meeting link |
| Fully automatic schedule generation with no admin correction | Organizers must retain control over final program placement |
| Unrestricted executable or arbitrary file uploads in chat | Chat attachments remain limited by explicit security rules |
| Full CMS for marketing/public pages | Structured conference content was sufficient for v1 |
